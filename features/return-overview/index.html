using namespace coro;

    Background

    One of the entry barriers for the coroutine function is to write a return type. The return type's promise_type must fulfills the Coroutine Promise Requirement (Actually it's coroutine_traits<R>::promise_type, but just let me skip the detail for now). To save your time, start with the article of Lewiss Baker. I also wrote about it with N4402, and with my talk in C++ Korea (see the page 60). In my opinion, N4402 was short enough and explains well about the role of those requirements.

    Contents

    Currently, this library provides 2 types.

    #include <coroutine/return.h>
using namespace coro;
    • no_return
    • frame

    You may noticed that there is no task<T>. I don't have a plan to support the type for 2 reasons.

    • To encourage use of https://github.com/lewissbaker/cppcoro which already implemented it
    • To encourage use of the suspension(co_await and co_yield) insead of the return(co_return)

    In my perspective, the coroutine is a routine with multiple entry points and multiple suspend points. Therefore we should be familiar with the suspension and have concern about its usage rather than just using return(the last suspension).
